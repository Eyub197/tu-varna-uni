How the Direction Vector Spiral Algorithm Works
===============================================

Instead of using 4 while loops that shrink the boundaries of the matrix (the typical ChatGPT solution), this algorithm uses a "Walk and Turn" approach. It mimics how a person might actually walk in a spiral room until they get stuck.

Core Concept: Direction Vectors
-------------------------------
We define the movement using two arrays: `dr` (direction row) and `dc` (direction column).
By combining an index from `dr` and `dc`, we can step in a specific direction.

For Matrix A (horizontal spiral starting to the right):
int dr[] = {0, 1, 0, -1};
int dc[] = {1, 0, -1, 0};

Let's break down the directions based on an index `dir` (from 0 to 3):
- dir = 0 (Right): Row changes by 0, Col changes by +1 -> (0, 1)
- dir = 1 (Down) : Row changes by +1, Col changes by 0 -> (1, 0)
- dir = 2 (Left) : Row changes by 0, Col changes by -1 -> (0, -1)
- dir = 3 (Up)   : Row changes by -1, Col changes by 0 -> (-1, 0)

For Matrix B (vertical spiral starting downwards):
We just swap the order of the vectors so we start downwards, then go right, then up, then left:
int dr[] = {1, 0, -1, 0};
int dc[] = {0, 1, 0, -1};


Step-by-Step Execution:
-----------------------
1. Preparation:
   We fill the entire matrix with 0s. This is crucial because it allows us to know which cells we have already visited (any cell that is no longer 0 is already filled).

2. Starting Point:
   We start at row `r = 0`, column `c = 0`. Our initial direction index is `dir = 0`. We loop through numbers `num` from 1 up to `n*n` (which is 25 for a 5x5 matrix).

3. Placing the Number:
   In each loop iteration, we write our current `num` into `a[r][c]`.

4. Looking Ahead (The "Walk"):
   We peek at the next cell we *would* step on if we kept walking straight in our current direction.
   nextR = r + dr[dir];
   nextC = c + dc[dir];

5. The Wall Check (The "Turn"):
   We check if our next step is valid. A step is INVALID if:
   - We hit the bottom or top wall: `nextR < 0` or `nextR >= n`
   - We hit the left or right wall: `nextC < 0` or `nextC >= n`
   - We hit our own path: `a[nextR][nextC] != 0` (this means we already visited that cell)

   If ANY of those conditions are true, we must turn!
   Think of `dir` (direction) like a 4-point compass numbered 0 to 3.
   To turn right mathematically, we add 1 to our current direction (`dir + 1`).
   - Facing Right (0) + 1 = Down (1)
   - Facing Down (1) + 1 = Left (2)
   - Facing Left (2) + 1 = Up (3)
   - Facing Up (3) + 1 = 4? Wait, there is no direction 4! We want to go back to 0 (Right).

   This is what the modulo `%` (remainder) operator does. When we do `% 4`, it wraps any number 4 or higher back to 0.
   - 3 % 4 = 3
   - 4 % 4 = 0 (4 divided by 4 leaves a remainder of 0)

   So, `dir = (dir + 1) % 4` is a simple mathematical trick to say: "Turn to the next direction, but if you go past 3, immediately loop back to 0."

   Once we turn, we recalculate our next step based on the NEW direction.

 Taking the Step:
   Finally, we update our current position `r` and `c` to the `nextR` and `nextC` that we calculated, and the loop repeats for the next number.

Why is this better to present?
------------------------------
It's much more elegant and uses a technique often taught in competitive programming or game development (using vectors for movement) instead of brute-forcing coordinate limits. It shows a deeper understanding of grid traversal.
